# My Control

A simple Android app that toggles **Wi-Fi** and **Mobile Data** on a daily schedule.

- Default: OFF at **23:00**, ON at **07:00**, every day.
- Works on **OnePlus 13R** (OxygenOS, Android 15) and most Android 8+ devices.
- Two modes:
  - **Reminder mode (default)** — fires a notification and opens the Wi-Fi / Mobile-Data settings panel at the scheduled time. You tap the toggle yourself. Reliable on every phone.
  - **Auto-tap mode (opt-in)** — uses Android's Accessibility service to open Quick Settings and tap the tiles automatically. No user interaction needed.

## Get the APK (GitHub Actions build)

You don't need Android Studio. GitHub will build the APK for you.

1. Sign in to GitHub (https://github.com). Create a free account if you don't have one.
2. Click the **+** icon (top right) → **New repository**. Name it `MyControl`. Set it to **Private**. Don't add a README, .gitignore, or license — leave them all unchecked. Click **Create repository**.
3. On the next page, click **uploading an existing file** (it's a link in the middle of the page). Drag the entire contents of this folder into the upload area (or click "choose your files"). Make sure you upload the `.github` folder too — if your computer hides dotfolders, enable hidden file display first.
4. Scroll down and click **Commit changes**.
5. Click the **Actions** tab at the top of the repo. A workflow named "Build APK" will already be running (yellow circle). Wait 3–5 minutes for it to finish (green check).
6. Click into the completed run. At the bottom under **Artifacts**, click **MyControl-debug-apk** to download a zip.
7. Unzip it. Inside you'll find `app-debug.apk`.

## Install on your OnePlus 13R

1. Transfer `app-debug.apk` to your phone (email, Google Drive, USB cable — any method).
2. On the phone, open your File Manager and tap the APK.
3. Android will say "your phone isn't allowed to install unknown apps from this source." Tap **Settings** → toggle on **Allow from this source**.
4. Back out, tap the APK again → **Install** → **Open**.

## First-run setup

When you open the app the first time:

1. Set your **OFF time** and **ON time** (defaults are 23:00 / 07:00).
2. Choose what to toggle: **Wi-Fi**, **Mobile data**, or both.
3. Tap **Enable schedule**. The app will:
   - Ask for the **exact alarm** permission (Android 12+). Allow it.
   - Ask for **notification** permission (Android 13+). Allow it.
4. Tap **Save schedule**.

That's it for reminder mode. You'll get a notification at the scheduled times, and the Wi-Fi/Mobile-Data panel will pop up — tap the toggle, done.

### Optional: turn on auto-tap mode

If you want fully hands-off toggling:

1. Inside the app, turn on **Auto-tap mode (Accessibility)**.
2. Tap **Open Accessibility settings**.
3. In the system Accessibility list, scroll down to **Installed apps** (or similar — on OxygenOS it's usually under "Downloaded apps"). Find **My Control auto-toggle** and turn it on. You'll see a system warning about what accessibility services can do — confirm.
4. Come back to the app. The status line should now say "Accessibility service: enabled."

### OnePlus 13R battery-optimization tip

OxygenOS aggressively kills background apps. To make sure scheduled alarms always fire:

1. Long-press the My Control icon → **App info**.
2. Tap **Battery** → set to **Don't optimize** (or "Unrestricted").

Without this, OxygenOS may delay or skip alarms when the phone is asleep.

## Building locally (optional)

If you'd rather build on your own computer:

1. Install Android Studio (https://developer.android.com/studio).
2. Open this folder as a project. Android Studio will download the Android SDK and Gradle automatically on first build.
3. **Note on the Gradle wrapper:** this repo ships `gradlew` and `gradlew.bat` but **not** `gradle/wrapper/gradle-wrapper.jar` (it's a binary that has to be generated by Gradle itself). When you first open the project, Android Studio will either auto-create it during Sync or prompt you to "Use Gradle wrapper" — accept it. If you'd rather get the jar directly without Android Studio, run the GitHub Actions build once; the workflow uploads the generated `gradle-wrapper.jar` as a separate artifact you can download and commit back into your repo.
4. Connect your phone via USB (with USB debugging enabled in Developer Options), or use the Android Studio "Build → Build Bundle(s) / APK(s) → Build APK(s)" menu to produce an APK without a phone.
